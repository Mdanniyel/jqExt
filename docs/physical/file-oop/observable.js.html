function addListener

public void addListener(String eventName, Function handler, [Object scope], [Object args])
public void addListener(Object options)
public void addListener(Object options)
Add listener to this object. There are 3 parameter forms, see below for detailed parameter information. Examples: 
   //Bind single event to handler
   obj.on('someEvent', handlerFunc, scope, [arg1, arg2, arg3]);

   //Bind multiple events to the same handler
   obj.on('event1 event2 event3', handlerFunc, scope, [arg1, arg2, arg3]);

   //Bind multiple events to different handles with the same scope and bind args:
   obj.on({
    event1: handlerFunc1,
    event2: handlerFunc2,
    scope: sameScopeObj,
    args: [1,2,3]
   });

   //Bind multiple events to different handles with various scope and bind args:
   obj.on({
    event1: { fn: handlerFunc1, scope: otherObj1, args: [1,2,3] },
    event2: { fn: handlerFunc2, scope: otherObj2, args: [4,5,6] }
   });
Parameters:
Classic
eventName - name of the event (or multiple events names separated by space)
handler - function to invoke when event is fired
[scope] - the scope to invoke handler function in. If not specified or null, defaults to the object firing the event
[args] - arguments to pass to the handler function as part of the event object (bindArgs property of the event object) 
   //Bind single event to handler: someEvent of obj will be bound to handlerFunc which will be
   //invoked in scope scope, and event object bindArgs property will be [arg1, arg2, arg3]
   obj.on('someEvent', handlerFunc, scope, [arg1, arg2, arg3]);

   //Bind multiple events to the same handler
   obj.on('event1 event2 event3', handlerFunc, scope, [arg1, arg2, arg3]);
Config object - type 1
options - configuration hash with keys being event names and values handler functions. In addition, scope and args keys can be specified.
  • Function eventName1 - key is the event name, value is handler function
  • [ Object scope ] - the scope to invoke all handler functions in. If not specified or null, defaults to the object firing the event
  • [ Object args ] - arguments to pass to all handler functions as part of the event object (bindArgs property of the event object) 
       //Bind multiple events to different handles with the same scope and bind args:
   obj.on({
    event1: handlerFunc1,
    event2: handlerFunc2,
    scope: sameScopeObj,
    args: [1,2,3]
   });
Config object - type 2
options - configuration hash with keys being event names and values configuration objects. In addition, scope and args keys can be specified.
  • Function eventName1 - key is the event name, value is configuration object with the following keys:
    • Function fn - function to invoke when event is fired
    • [ Object scope ] - scope to invoke fn in
    • [ Object args ] - bind args to pass as part of event object
  • [ Object scope ] - the scope to invoke all handler functions in. If not specified or null, defaults to the object firing the event
  • [ Object args ] - arguments to pass to all handler functions as part of the event object (bindArgs property of the event object) 
       //Bind multiple events to different handles with various scope and bind args:
   obj.on({
    event1: { fn: handlerFunc1, scope: otherObj1, args: [1,2,3] },
    event2: { fn: handlerFunc2, scope: otherObj2, args: [4,5,6] }
   });

function fireListener

public Array fireListener(String eventName, args...)
Fire all registered listeners for given event name. Note if one of the listeners throws an error, other listeners don't get invoked, and error is propagated. If your listener can throw error, but you want to allow normal excecution, wrap your listener code in try/catch.
Parameters:
eventName - event name to fire listeners for
args... - arguments to pass to handler functions after the event object
Returns:
array of listeners return values (in the order of invokation) or false if excecution has been terminated by returning $.ext.$break from handler function (listener)

function removeListener

public Object removeListener()
Remove listeners from this object. Usages: - Remove specific listener for event: obj.removeListener('someEvent', eventListener) - Remove all listeners for event: obj.removeListener('someEvent') - Remove all events: myObject.un() - Remove multiple listeners for events (if listener value for event is null - all listeners for this event will be unbound): obj.un({event1: event1Listener, event2: event2Listener, event3: null, ...}); - Remove all listeners for multiple events: myObject.un('event1 event2 event3');
